## 배열

- **같은 타입**의 변수들이 **연속된 메모리**로 저장되어 있는 집합이다.
- 중복을 허용하고, 순서가 있다.

### 시간 복잡도

| 접근 | 탐색 |          삽입          |          삭제          |
| :--: | :--: | :--------------------: | :--------------------: |
| O(1) | O(n) | 끝 : O(1), 중간 : O(n) | 끝 : O(1), 중간 : O(n) |

- 접근은 `address` = `base_address` + `(index × element_size)` 이기에 **인덱스만 알면** 해당 위치로 바로 접근이 가능하다.
- 탐색은 순차적으로 탐색한다. 다만, 정렬되어 있다면 **이진 탐색법**이 가능하기에 `O(logn)`이다. 
- 삽입과 삭제는 **사이즈를 알기에** 마지막 위치에 바로 접근이 가능하나, 중간은 해당 위치 이후의 데이터를 **순차적으로 밀거나 댕겨야** 한다.
- 임의의 인덱스에 해당하는 데이터에 접근하는 **랜덤 접근에 유용하다**.

## 스택

- **후입선출(LIFO, Last In First Out)** 구조의 자료구조이다.
- 가장 **최근에 삽입된 데이터가 가장 먼저 제거**된다.
- 데이터의 삽입과 삭제는 **한쪽 끝(top)** 에서만 이루어진다.

### 시간 복잡도

| 접근(peek) | 탐색 | 삽입(push) | 삭제(pop) |
| :--------: | :--: | :--------: | :-------: |
|    O(1)    | O(n) |    O(1)    |   O(1)    |

- 스택은 `top` **위치만 관리**하기 때문에 접근(peek)과 삽입(push), 삭제(pop)는 항상 `O(1)` 이다.
- 임의의 위치에 접근하는 **랜덤 접근은 지원하지 않는다.** 스택의 목적이 최근 데이터를 처리하기 위함이기 때문이다.
- 탐색을 하려면 **위에서부터 순차적**으로 확인해야 하기 때문에 `O(n)` 이다.
- 배열이나 연결 리스트를 이용해 구현할 수 있으며, 대부분 `top` **포인터(또는 index)**로 현재 위치를 관리한다.
